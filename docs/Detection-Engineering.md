# Detection Engineering

## Overview

This document describes how endpoint telemetry generated during the Atomic Red Team simulations was collected, analysed, and investigated throughout the EDR Detection and Incident Response Lab.

Rather than creating custom detection rules, this project focused on understanding how endpoint telemetry generated by common ATT&CK techniques can be investigated using LimaCharlie and Sysmon.

---

# Objective

The objectives of this phase were to:

- Generate endpoint telemetry using Atomic Red Team.
- Observe how Windows records attacker behaviour.
- Investigate endpoint activity using LimaCharlie.
- Understand how common attacker techniques appear within endpoint telemetry.
- Build practical incident investigation skills.

---

# Telemetry Sources

| Source | Purpose |
|---------|---------|
| Sysmon | Generates detailed endpoint telemetry |
| Windows Event Logs | Stores endpoint activity |
| LimaCharlie Sensor | Collects and forwards telemetry |
| LimaCharlie Cloud | Endpoint investigation platform |

Examples of monitored activity included:

- Process Creation
- Process Termination
- File Creation
- Network Connections
- Registry Activity
- Driver Loading
- Image Loading

This telemetry formed the foundation for every investigation performed during the lab.

---

# Investigation Methodology

Every ATT&CK technique followed the same investigation process.

1. Execute the Atomic Red Team test.
2. Confirm successful execution.
3. Open LimaCharlie.
4. Locate the generated endpoint activity.
5. Review the timeline.
6. Inspect the event details.
7. Analyse the process information.
8. Document the findings.

One notable example was **T1047 – WMI Execute Local Process**, where the Atomic test failed because the legacy **WMIC** utility is no longer available by default on modern Windows 11 systems. Although the simulation did not complete successfully, endpoint telemetry was still generated and investigated.

---

# Investigation Focus Areas

Each investigation focused on the following endpoint artefacts.

## Process Information

- Process Name
- Parent Process
- Process ID (PID)
- Parent Process ID (PPID)

## Command-Line Analysis

Command-line arguments were reviewed to understand how the technique executed and what behaviour was generated on the endpoint.

## Process Relationships

Special attention was given to parent-child process relationships to understand how processes were spawned during execution.

## Timeline Analysis

LimaCharlie Timeline was used to correlate endpoint events generated during each Atomic Red Team simulation.

## Endpoint Context

Whenever available, additional endpoint context was reviewed, including:

- Executable path
- User context
- Timestamp
- Related endpoint events

# Lessons for SOC Analysts

This project demonstrates that effective endpoint investigations rely on understanding normal endpoint behaviour before analysing suspicious activity.

The combination of Sysmon telemetry and LimaCharlie provides valuable visibility into:

- Process execution
- Endpoint behaviour
- Command-line activity
- Timeline correlation
- Process relationships

These artefacts form the foundation of many endpoint investigations performed by SOC analysts.

# Conclusion

The objective of this project was to understand how attacker techniques appear from a defender's perspective rather than to create production detection content.

By executing ATT&CK techniques in a controlled lab and analysing the resulting telemetry inside LimaCharlie, the project provides practical experience with endpoint investigation and detection analysis that aligns with common SOC analyst workflows.
