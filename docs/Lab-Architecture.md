# Lab Architecture

## Overview

The EDR Detection and Incident Response Lab was designed to simulate a monitored Windows endpoint capable of generating endpoint telemetry for detection, investigation, and incident response.

The lab combines virtualization, endpoint telemetry, Endpoint Detection and Response (EDR), and attack simulation using Atomic Red Team.

The objective was to observe how attacker techniques appear within endpoint telemetry and investigate the resulting activity using LimaCharlie.

# Components

## Oracle VirtualBox

Oracle VirtualBox was used as the hypervisor to create an isolated Windows environment.

Benefits:

- Safe attack simulation
- Snapshot support
- Isolation from the host system
- Easy recovery after testing

---

## Windows 11 Virtual Machine

The Windows 11 virtual machine acted as the endpoint under investigation.

The VM hosted:

- Sysmon
- LimaCharlie Sensor
- Atomic Red Team
- PowerShell

Every simulated ATT&CK technique was executed on this endpoint.

---

## Sysmon

Sysmon enhanced Windows logging by recording detailed endpoint telemetry.

Examples of monitored activity included:

- Process Creation
- Process Termination
- Network Connections
- File Creation
- Registry Activity
- Driver Loading

This telemetry formed the foundation for endpoint investigations.

---

## Windows Event Logs

Sysmon events were written to the Windows Event Log.

Location:

```text
Applications and Services Logs
    └── Microsoft
          └── Windows
                └── Sysmon
                      └── Operational
```

These logs were collected by the LimaCharlie sensor.

---

## LimaCharlie Sensor

The LimaCharlie Windows Sensor continuously monitored endpoint activity and securely transmitted telemetry to the LimaCharlie cloud platform.

The sensor collected:

- Process execution
- Command-line arguments
- File activity
- Network activity
- User context
- Process hashes

---

## LimaCharlie Cloud Platform

The cloud platform provided visibility into endpoint activity.

The following investigation views were used throughout the project:

- Timeline
- Live Feed
- Event Details
- Process Investigation

These views were used to analyse telemetry generated by each Atomic Red Team technique.

---

## Atomic Red Team

Atomic Red Team was used to safely simulate MITRE ATT&CK techniques.

Each Atomic test generated realistic endpoint telemetry without requiring live malware.

The project documented the following ATT&CK techniques:

- T1059.001 – PowerShell
- T1218.015 – Signed Binary Proxy Execution
- T1105 – Ingress Tool Transfer
- T1047 – Windows Management Instrumentation
- T1036 – Masquerading
- T1070 – Indicator Removal
- T1082 – System Information Discovery
- T1016 – System Network Configuration Discovery
- T1083 – File and Directory Discovery
- T1005 – Data from Local System
- T1027 – Obfuscated/Compressed Files and Information

---

# Telemetry Flow

The overall telemetry flow throughout the project followed this sequence:

1. Atomic Red Team executed an ATT&CK technique.
2. Windows executed the associated process.
3. Sysmon recorded endpoint activity.
4. Windows Event Logs stored the telemetry.
5. LimaCharlie Sensor collected the events.
6. Telemetry was uploaded to the LimaCharlie Cloud Platform.
7. Endpoint activity was investigated using Timeline and Event Details.
