# MITRE ATT&CK Mapping

## Overview

This project uses the MITRE ATT&CK framework to simulate and investigate common adversary behaviours on a monitored Windows endpoint.

Each Atomic Red Team test was selected to represent a specific ATT&CK technique. The resulting telemetry was collected by Sysmon, forwarded to LimaCharlie, and investigated through the EDR platform.

The purpose of this mapping is to demonstrate how common adversary techniques can be observed from a defender's perspective through endpoint telemetry and investigation, rather than treating ATT&CK as a simple checklist.

# Technique Summary

## T1059.001 – PowerShell

Simulated PowerShell execution using Atomic Red Team to observe endpoint telemetry generated by PowerShell-based command execution.

## T1218.015 – Signed Binary Proxy Execution

Simulated execution using the trusted Windows binary **rundll32.exe** to demonstrate abuse of legitimate Microsoft-signed binaries.

## T1105 – Ingress Tool Transfer

Simulated downloading a file to the endpoint to represent attacker tool or payload transfer after initial compromise.

## T1047 – Windows Management Instrumentation

Attempted local process execution using WMIC. The execution did not complete because **WMIC is no longer available by default on modern Windows 11 systems**, demonstrating how operating system versions can influence attack simulations.

## T1036 – Masquerading

Simulated disguising executable activity to resemble legitimate software while analysing the generated endpoint telemetry.

## T1070 – Indicator Removal

Simulated removal of forensic artefacts after execution to observe how cleanup activity appears within endpoint telemetry.

## T1082 – System Information Discovery

Collected operating system version information using the Windows **ver** command.

## T1016 – System Network Configuration Discovery

Executed Windows networking utilities including ipconfig, netsh, arp, nbtstat, and net config to simulate attacker network reconnaissance.

## T1083 – File and Directory Discovery

Performed recursive directory enumeration using built-in Windows commands to identify files and folders of interest.

## T1005 – Data from Local System

Simulated collecting files from the local system for potential staging or exfiltration.

## T1027 – Obfuscated/Compressed Files and Information

Executed an encoded PowerShell command to simulate command obfuscation commonly observed during real-world attacks.

# Conclusion

This project demonstrates practical application of the MITRE ATT&CK framework within a controlled Windows endpoint detection lab.

Rather than treating ATT&CK as a list of techniques, the project focuses on understanding how each technique generates endpoint telemetry and how that activity can be investigated using LimaCharlie EDR.

The individual technique documentation provides the detailed execution evidence, investigation process, and observations for each ATT&CK technique covered in this repository.
